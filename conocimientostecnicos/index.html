A) Infraestructura y Orquestación

  1. Docker (https://www.docker.com/)
    Tecnología de contenedores que permite empaquetar aplicaciones y sus dependencias en unidades estandarizadas. Aunque no se cita nominalmente en el texto, constituye la base técnica necesaria para el aislamiento y despliegue de los servicios en el clúster de Kubernetes.

  2. Kubernetes (https://kubernetes.io/)
    Sistema de orquestación de código abierto diseñado para automatizar el despliegue, el escalado y la gestión de aplicaciones en contenedores dentro de la infraestructura del proyecto.

  3. Linux (https://www.linux.org/)
    Sistema operativo fundamental de tipo Unix que actúa como entorno de ejecución base para todos los nodos del clúster y la totalidad de los servicios del espacio de datos.

  4. AWS (Amazon Web Services) (https://aws.amazon.com/es/)
    Proveedor de servicios en la nube utilizado para alojar la infraestructura del sistema, proporcionando los recursos necesarios de computación, almacenamiento y red.

  5. NFS (Network File System) Provider (https://nfs.sourceforge.net/)
    Protocolo de sistema de archivos de red que permite a los distintos componentes de Kubernetes acceder de forma concurrente a volúmenes de almacenamiento compartido persistente.

  6. S3 (Simple Storage Service) (https://aws.amazon.com/es/s3/)
    Servicio de almacenamiento de objetos escalable empleado para guardar grandes volúmenes de datos y activos lingüísticos de forma segura, duradera y accesible.

B) DevOps y Automatización

  1. Terraform (https://www.terraform.io/)
    Herramienta de infraestructura como código utilizada para definir y aprovisionar de manera automática y reproducible todos los recursos de infraestructura del proyecto.

  2. OpenTofu (https://opentofu.org/)
    Bifurcación de código abierto de Terraform empleada para la gestión, el despliegue y el mantenimiento de los recursos de infraestructura en entornos cloud.

  3. ArgoCD (Argo Continuous Delivery) (https://argo-cd.readthedocs.io/)
    Herramienta de entrega continua declarativa para Kubernetes que sigue la metodología GitOps para sincronizar el estado del clúster con los repositorios de configuración.

  4. Argo Applications (https://argo-cd.readthedocs.io/en/stable/operator-manual/declarative-setup/)
    Recurso de ArgoCD que permite agrupar, definir y gestionar múltiples componentes de Kubernetes como una única unidad lógica de aplicación para facilitar su administración.

  5. Helm (https://helm.sh/)
    Gestor de paquetes para Kubernetes que utiliza plantillas de configuración denominadas charts para automatizar la instalación y actualización de aplicaciones complejas.

  6. Git (https://git-scm.com/)
    Sistema de control de versiones distribuido que funciona como la fuente de verdad única para el código fuente y las definiciones de infraestructura bajo el modelo GitOps.

C) Backend y Gestión de Contenidos

  1. Drupal 11 (https://www.drupal.org/)
    Sistema de gestión de contenidos utilizado para construir el portal del espacio de datos, integrando la administración de usuarios, flujos de trabajo y activos.

  2. PHP (Hypertext Preprocessor) (https://www.php.net/)
    Lenguaje de programación del lado del servidor en el que está desarrollado el núcleo de Drupal y que se emplea para implementar la lógica de negocio personalizada.

  3. OnlyOffice Document Server (https://www.onlyoffice.com/)
    Suite ofimática en línea integrada para proporcionar capacidades de visualización y edición colaborativa de documentos de texto y hojas de cálculo dentro de la plataforma.

  4. Mautic (https://www.mautic.org/)
    Plataforma de automatización de marketing de código abierto utilizada para la gestión de comunicaciones, notificaciones y campañas de difusión hacia los usuarios del espacio.

  5. ECA (Event - Condition - Action) (https://www.drupal.org/project/eca)
    Motor de automatización para Drupal que permite configurar de forma gráfica flujos de trabajo complejos basados en disparadores, condiciones y acciones automatizadas.

  6. Document library (https://www.drupal.org/project/document_library)
    Módulo especializado dentro de la arquitectura de Drupal diseñado para la organización, categorización y gestión eficiente de la biblioteca de recursos digitales.

D) Frontend y Diseño

  1. NextJS (https://nextjs.org/)
    Framework de desarrollo web basado en React que permite el renderizado en el lado del servidor para optimizar el rendimiento y la visibilidad en motores de búsqueda.

  2. React (https://react.dev/)
    Biblioteca de JavaScript orientada a la creación de interfaces de usuario dinámicas mediante componentes reutilizables que gestionan su propio estado de forma eficiente.

  3. Tailwind CSS (Cascading Style Sheets) (https://tailwindcss.com/)
    Framework de estilos orientado a utilidades que permite diseñar interfaces personalizadas y responsivas de forma rápida directamente desde el código frontal.

  4. DaisyUI (User Interface) (https://daisyui.com/)
    Colección de componentes visuales predefinidos para Tailwind CSS que agilizan el desarrollo de la interfaz de usuario manteniendo una estética consistente.

  5. Figma (https://www.figma.com/)
    Herramienta de diseño y prototipado colaborativo en la nube utilizada para definir la experiencia de usuario y crear los modelos visuales de alta fidelidad.

  6. Whimsical (https://whimsical.com/)
    Plataforma de trabajo visual empleada para la elaboración de diagramas de flujo, esquemas de arquitectura de información y mapas conceptuales durante el diseño.

E) Bases de Datos y Mensajería

  1. PostgreSQL (https://www.postgresql.org/)
    Sistema de gestión de bases de datos relacionales robusto y de alto rendimiento utilizado para el almacenamiento persistente de la información de los servicios.

  2. Kafka (https://kafka.apache.org/)
    Plataforma distribuida de transmisión de eventos que permite el procesamiento y la comunicación de flujos de datos en tiempo real entre microservicios.

  3. Kafka Kraft (https://developer.confluent.io/learn/kraft/)
    Protocolo de consenso que permite la ejecución de clústeres de Kafka sin dependencia de ZooKeeper, simplificando la arquitectura y la gestión de la mensajería.

  4. Elasticsearch (https://www.elastic.co/elasticsearch/)
    Motor de búsqueda y analítica distribuido que permite realizar consultas de texto completo rápidas sobre grandes volúmenes de metadatos y registros.

  5. Kibana (https://www.elastic.co/kibana/)
    Herramienta de visualización que proporciona interfaces gráficas para explorar, analizar y monitorizar los datos y registros almacenados en Elasticsearch.

F) Observabilidad y Monitorización

  1. Prometheus (https://prometheus.io/)
    Sistema de monitorización y alerta que recopila métricas de series temporales de los componentes del sistema para supervisar su rendimiento y disponibilidad.

  2. Grafana (https://grafana.com/)
    Plataforma de analítica y visualización que permite construir paneles interactivos a partir de las métricas de Prometheus para facilitar la toma de decisiones.

  3. Loki (https://grafana.com/oss/loki/)
    Sistema de agregación de registros diseñado para ser eficiente y trabajar de forma integrada con Prometheus en entornos de orquestación de contenedores.

  4. Jaeger (https://www.jaegertracing.io/)
    Herramienta de código abierto para el rastreo distribuido de extremo a extremo, utilizada para monitorizar y depurar transacciones en arquitecturas complejas.

  5. AlertManager (https://prometheus.io/docs/alerting/latest/alertmanager/)
    Componente encargado de gestionar el ciclo de vida de las alertas generadas por Prometheus, ocupándose de su deduplicación, agrupación y envío a canales externos.

  6. OTel (OpenTelemetry) (https://opentelemetry.io/)
    Conjunto de estándares y herramientas diseñado para la generación, recolección y exportación de datos de telemetría de forma neutral respecto al proveedor.

  7. ECK (Elastic Cloud on Kubernetes) (https://www.elastic.co/guide/en/cloud-on-k8s/current/index.html)
    Operador que automatiza y simplifica la gestión de clústeres de Elasticsearch, Kibana y otros componentes de Elastic dentro de entornos de Kubernetes.

G) Soberanía y Espacios de Datos

  1. EDC (Eclipse Dataspace Connector) (https://www.google.com/search?q=https://eclipse-edc.github.io/docs/)
    Componente tecnológico fundamental que habilita el intercambio de datos soberano, seguro e interoperable bajo los estándares de los espacios de datos europeos.

  2. SIMPL-Open (https://www.google.com/search?q=https://simpl-open.eu/)
    Framework impulsado por la Unión Europea que proporciona la base tecnológica para el despliegue de infraestructuras de datos federadas y soberanas.

  3. SDK (Software Development Kit) (https://github.com/eclipse-edc/Connector)
    Kit que incluye librerías, herramientas y documentación para facilitar a los desarrolladores la integración de sistemas externos con el conector del espacio.

  4. EDC Management UI (User Interface) (https://www.google.com/search?q=https://github.com/eclipse-edc/ManagementUI)
    Interfaz gráfica de usuario dedicada a la administración simplificada de las operaciones del conector, como la gestión de contratos y políticas.

  5. QuickSearch (https://gnoss.com/)
    Funcionalidad de búsqueda rápida integrada en el portal que permite localizar activos lingüísticos dentro del catálogo semántico de forma ágil.

  6. SD (Service Description) Creation Tool (https://www.google.com/search?q=https://simpl-open.eu/)
    Herramienta diseñada para automatizar y facilitar la creación de descripciones de servicio normalizadas para los participantes del ecosistema.

  7. SD (Service Description) Schema Management Service (https://www.google.com/search?q=https://simpl-open.eu/)
    Servicio encargado de la gobernanza y mantenimiento de los esquemas de datos utilizados para describir técnicamente los recursos del espacio.

H) Estándares y Protocolos

  1. LanguageDCAT-AP (Language Data Catalog Vocabulary - Application Profile) (https://www.google.com/search?q=https://joinup.ec.europa.eu/collection/semantic-interoperability-community-semic/solution/dcat-application-profile-data-portals-europe)
    Extensión del estándar DCAT-AP diseñada específicamente para la descripción detallada y normalizada de activos de lenguaje y recursos lingüísticos.

  2. DCAT (Data Catalog Vocabulary) v3 (https://www.w3.org/TR/vocab-dcat-3/)
    Vocabulario estándar para la descripción de catálogos de datos que facilita la interoperabilidad semántica entre distintas plataformas de publicación.

  3. DCAT-AP (Data Catalog Vocabulary - Application Profile) (https://www.google.com/search?q=https://joinup.ec.europa.eu/collection/semantic-interoperability-community-semic/solution/dcat-application-profile-data-portals-europe)
    Perfil de aplicación del estándar DCAT definido para portales de datos abiertos en Europa, garantizando la compatibilidad de metadatos a nivel regional.

  4. MLDCAT-AP (Machine Learning Data Catalog Vocabulary - Application Profile) (https://joinup.ec.europa.eu/collection/semantic-interoperability-community-semic)
    Perfil de metadatos adaptado para describir recursos y modelos de aprendizaje automático, facilitando su descubrimiento en el ámbito de la IA.

  5. ODRL (Open Digital Rights Language) (https://www.w3.org/TR/odrl-model/)
    Lenguaje estándar para la expresión formal de políticas de acceso, licencias y reglas de uso que regulan el intercambio de activos de datos.

  6. DSP (Dataspace Protocol) (https://docs.internationaldataspaces.org/ids-knowledgebase/dataspace-protocol)
    Protocolo de interoperabilidad que define las interacciones técnicas entre conectores para el descubrimiento, la negociación contractual y la transferencia.

  7. DCP (Decentralized Claim Protocol) (https://www.google.com/search?q=https://github.com/eclipse-dataspace-protocol-base/DecentralizedClaimProtocol)
    Protocolo utilizado para gestionar la verificación de identidades y atributos de participantes de forma descentralizada dentro del espacio de datos.

  8. JSON-LD (JavaScript Object Notation for Linked Data) (https://json-ld.org/)
    Método de serialización para datos enlazados que permite añadir contexto semántico a los datos JSON de forma legible para máquinas.

  9. RDF (Resource Description Framework) (https://www.w3.org/RDF/)
    Modelo estándar para el intercambio de datos en la web basado en la representación de información mediante estructuras de sujeto-predicado-objeto.

  10. SKOS (Simple Knowledge Organization System) (https://www.w3.org/2004/02/skos/)
    Estándar diseñado para representar tesauros, taxonomías y esquemas de clasificación dentro del marco de la web semántica.

  11. Dublin Core (https://dublincore.org/)
    Conjunto de metadatos básicos y universales utilizado para la descripción genérica de cualquier tipo de recurso digital o físico.

  12. META-SHARE (http://www.meta-share.org/)
    Modelo de metadatos especializado en la descripción de recursos lingüísticos, integrado para facilitar la gestión técnica del corpus lingüístico.

  13. BPMN (Business Process Model and Notation) 2.0 (https://www.omg.org/spec/BPMN/2.0/)
    Estándar de modelización de procesos que utiliza una notación gráfica para definir formalmente la secuencia de actividades del espacio de datos.

  14. SHACL (Shapes Constraint Language) (https://www.w3.org/TR/shacl/)
    Lenguaje utilizado para validar la estructura y el contenido de grafos de datos RDF frente a un conjunto de formas y reglas definidas.

  15. TLS (Transport Layer Security) 1.2 o superior (https://datatracker.ietf.org/doc/html/rfc5246)
    Protocolo de seguridad criptográfico que garantiza la privacidad y la integridad de la información durante las transferencias entre conectores.

  16. OpenAPI 3.0 y 3.1 (https://www.openapis.org/)
    Especificaciones estándar para describir y documentar interfaces de programación de aplicaciones RESTful, facilitando su consumo automatizado.

  17. BCP47 (https://tools.ietf.org/html/bcp47)
    Estándar técnico utilizado para la creación de etiquetas de identificación de lenguajes, esencial para la clasificación del corpus multilingüe.

  18. JWT (JSON Web Token) (https://jwt.io/)
    Estándar compacto y seguro para la representación de afirmaciones que se transfieren entre dos partes, utilizado para la autenticación y autorización.

  19. REST (Representational State Transfer) (https://www.google.com/search?q=https://ics.uci.edu/~fielding/pubs/dissertation/rest_arch_style.htm)
    Estilo de arquitectura de software para sistemas distribuidos basado en el protocolo HTTP, utilizado para exponer las funciones del SDK.

  20. JSON (JavaScript Object Notation) (https://www.json.org/)
    Formato de intercambio de datos ligero, basado en texto y fácil de leer para humanos y máquinas, utilizado como estándar en todas las APIs.

I) Seguridad e Identidad

  1. Keycloak (https://www.keycloak.org/)
    Solución de gestión de identidad y acceso de código abierto que proporciona autenticación de factor múltiple y gestión centralizada de usuarios.

  2. OAuth (Open Authorization) 2.0 (https://oauth.net/2/)
    Protocolo estándar de autorización que permite a las aplicaciones obtener acceso limitado a cuentas de usuario de forma segura.

  3. OIDC (OpenID Connect) (https://openid.net/connect/)
    Capa de identidad sencilla construida sobre el protocolo OAuth 2.0 que permite la verificación de la identidad del usuario final.

  4. LDAP (Lightweight Directory Access Protocol) (https://ldap.com/)
    Protocolo estándar utilizado para acceder y gestionar servicios de información de directorios y bases de datos de usuarios corporativos.

  5. AD (Active Directory) (https://learn.microsoft.com/en-us/windows-server/identity/ad-ds/active-directory-domain-services)
    Servicio de directorio de red desarrollado por Microsoft para la gestión centralizada de identidades y permisos en entornos empresariales.

  6. Vault (https://www.vaultproject.io/)
    Herramienta diseñada para gestionar secretos y proteger datos sensibles mediante el control de acceso a tokens, contraseñas y claves criptográficas.

  7. HSM (Hardware Security Module) (https://en.wikipedia.org/wiki/Hardware_security_module)
    Dispositivo físico de seguridad encargado de la generación, almacenamiento y gestión segura de claves criptográficas dentro de un entorno protegido.

  8. Cloud KSM (Keeper Secrets Manager) (https://www.keepersecurity.com/es_ES/secrets-manager.html)
    Plataforma basada en la nube para la gestión y protección de secretos, credenciales de infraestructura y claves de acceso de forma escalable.

  9. AWS (Amazon Web Services) Secret Manager (https://aws.amazon.com/es/secrets-manager/)
    Servicio gestionado que ayuda para proteger los secretos necesarios para acceder a aplicaciones, servicios y recursos de TI en la nube de Amazon.

  10. OpenBao (https://openbao.org/)
    Solución de código abierto para la gestión de secretos y protección de datos sensibles, originada como alternativa comunitaria a Vault.

  11. Cert Manager (https://cert-manager.io/)
    Controlador nativo de Kubernetes que automatiza la gestión de certificados TLS, encargándose de su emisión y renovación periódica.

  12. EJBCA (Enterprise Java Bean Certificate Authority) (https://www.ejbca.org/)
    Plataforma de software de infraestructura de clave pública robusta utilizada para la emisión y gestión de certificados digitales a gran escala.

  13. Certificados X.509 (https://datatracker.ietf.org/doc/html/rfc5280)
    Estándar técnico que define el formato de los certificados de clave pública utilizados para establecer conexiones seguras mediante mTLS.

  14. DAT (Dynamic Attribute Tokens) (https://www.google.com/search?q=https://docs.internationaldataspaces.org/)
    Tokens que contienen atributos dinámicos y verificados de un participante, permitiendo decisiones de acceso basadas en el contexto de seguridad.

  15. VC (Verifiable Credentials) (https://www.w3.org/TR/vc-data-model/)
    Estándar para credenciales digitales que pueden ser validadas criptográficamente de forma segura, respetando la privacidad del usuario.

  16. Wallets (https://en.wikipedia.org/wiki/Digital_wallet)
    Aplicaciones o servicios encargados de almacenar, gestionar y presentar de forma segura las credenciales verificables y las claves privadas de los usuarios.

  17. ESO (External Secrets Operator) (https://external-secrets.io/)
    Herramienta de Kubernetes que permite integrar de forma segura gestores de secretos externos con el ecosistema de aplicaciones del clúster.

  18. Apisix Ingress (https://apisix.apache.org/)
    Controlador de entrada de alto rendimiento basado en Apache APISIX que actúa como puerta de enlace para gestionar y asegurar el tráfico de las APIs.

  19. SHA (Secure Hash Algorithm)-256 (https://en.wikipedia.org/wiki/SHA-2)
    Función criptográfica que genera un resumen único de 256 bits a partir de un conjunto de datos, utilizada para verificar la integridad de la información.

  20. ECDSA (Elliptic Curve Digital Signature Algorithm) (https://en.wikipedia.org/wiki/Elliptic_Curve_Digital_Signature_Algorithm)
    Algoritmo de firma digital basado en criptografía de curva elíptica que proporciona un alto nivel de seguridad con claves de menor longitud.

  21. mTLS (Mutual Transport Layer Security) (https://www.cloudflare.com/learning/access-management/what-is-mutual-tls/)
    Protocolo de seguridad en el que tanto el cliente como el servidor se autentican mutuamente mediante certificados digitales antes de comunicarse.

  22. Zero Trust (https://www.nist.gov/publications/zero-trust-architecture)
    Marco de seguridad basado en el principio de no confiar nunca por defecto en ningún usuario o dispositivo, verificando cada solicitud de acceso.
